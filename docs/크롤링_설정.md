# 크롤링 설정 (로컬 Python)

공모전·창업 데이터는 **Supabase Edge Function이나 GitHub Actions 없이**, 서버 PC 등에서 `crawl_server.py`만 실행해 Supabase에 `upsert`합니다.

자세한 실행 방법·환경 변수·파일 역할은 루트 **[README.md](../README.md)** 의 Python 절을 참고하세요.

---

## Realtime 즉시 반영 (선택)

프론트엔드가 `contests` 등 INSERT 시 자동 갱신하려면 **Replication**을 켜야 합니다.

**Supabase 대시보드** → Database → Replication → 해당 테이블 ON.

(이미 등록된 경우 `ALTER PUBLICATION supabase_realtime ADD TABLE contests` 실행 시 "already member" 오류가 나오면 정상입니다.)

`.env`에 `K_START_UP_SERVICE` 를 넣습니다. (공공데이터포털에서 발급한 값, **이미 URL 인코딩된 키면 그대로** 사용 — Python 코드에서 추가 인코딩하지 않음.)
