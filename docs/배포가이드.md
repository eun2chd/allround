## 2. 환경 변수

배포 시 반드시 설정할 환경 변수입니다.

| 변수명 | 필수 | 설명 |
|--------|------|------|
| `VITE_NTP_SUPABASE_URL` | ✅ | Supabase 프로젝트 URL (예: `https://xxxx.supabase.co`) |
| `VITE_NTP_SUPABASE_ANON_KEY` | ✅ | Supabase Anon Key (Settings → API) |
| `SUPABASE_SERVICE_ROLE_KEY` | ✅ | Supabase Service Role Key (RLS 우회, presence 등) |
| `FLASK_SECRET_KEY` | ✅ | Flask 세션/쿠키 암호화용 (32자 이상 랜덤 문자열 권장) |
| `PORT` | ❌ | 서버 포트 (기본 5000, Render/Railway 등은 자동 지정) |
| `VITE_NTP_DATABASE_URL` | ❌ | DB 연결 (일부 기능용, 선택) |
| `VITE_NTP_DATABASE_DIRECT_URL` | ❌ | Direct DB 연결 (선택) |

### SECRET_KEY 생성 예시

```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

로컬 테스트 시 `.env.example`을 `.env`로 복사 후 값은 채운 뒤 실행합니다.

## 3. Supabase 설정

### 3.1 Redirect URLs (인증)

**Authentication** → **URL Configuration** → **Redirect URLs**에 배포 URL 추가:

| 용도 | 예시 URL |
|------|----------|
| 비밀번호 재설정 | `https://yourdomain.com/reset-password` |
| 로컬 테스트 | `http://localhost:5000/reset-password` |

### 3.2 Site URL

**Authentication** → **URL Configuration** → **Site URL**을 배포 도메인으로 설정  
예: `https://yourdomain.com`

### 3.3 이메일 설정

- **Email Provider**: 기본 Supabase SMTP 또는 Resend/SendGrid 연동
- 회원가입 인증 메일, 비밀번호 재설정 메일이 발송되도록 설정 확인

## 4. 배포 플랫폼별 가이드

### 4.1 Render

1. [Render](https://render.com) → New → Web Service
2. GitHub 저장소 연결
3. 설정:
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `gunicorn -w 4 -b 0.0.0.0:$PORT app:app`
   - **Environment Variables**: 위 환경 변수 추가

4. Deploy

### 4.2 Railway

1. [Railway](https://railway.app) → New Project → Deploy from GitHub
2. 환경 변수 추가
3. **Settings** → **Networking** → **Generate Domain**으로 URL 생성
4. `PORT`는 Railway가 자동 주입

**Start Command** (필요 시):

```
gunicorn -w 4 -b 0.0.0.0:$PORT app:app
```

### 4.3 Fly.io

1. `flyctl auth login`
2. `flyctl launch` (프로젝트 루트에서)
3. `flyctl secrets set` 으로 환경 변수 설정:

```bash
flyctl secrets set VITE_NTP_SUPABASE_URL="https://xxxx.supabase.co"
flyctl secrets set VITE_NTP_SUPABASE_ANON_KEY="eyJ..."
flyctl secrets set SUPABASE_SERVICE_ROLE_KEY="eyJ..."
flyctl secrets set FLASK_SECRET_KEY="your-32-char-secret"
```

4. `fly.toml`에 `internal_port = 5000` 또는 `processes = ["app"]` 설정 후:
   - `[env]` 섹션에 `PORT = "5000"` 추가
   - `cmd` 예: `gunicorn -w 4 -b 0.0.0.0:5000 app:app`

5. `flyctl deploy`

### 4.4 Procfile (Heroku / Render / Railway 호환)

프로젝트 루트에 `Procfile`이 포함되어 있습니다:

```
web: gunicorn -w 4 -b 0.0.0.0:$PORT app:app
```

`$PORT`는 Heroku, Render, Railway 등이 자동 설정합니다. `.env.example`을 참고하여 환경 변수를 설정하세요.

## 5. 배포 후 점검

### 필수 확인 사항

| 항목 | 확인 방법 |
|------|-----------|
| 로그인 | 이메일/비밀번호 로그인 동작 |
| 회원가입 | 가입 → 이메일 인증 → 로그인 |
| 아이디/비밀번호 찾기 | find-account 페이지에서 동작 |
| 비밀번호 재설정 | 이메일 링크 → 새 비밀번호 설정 |
| 공모전 목록 | 메인 페이지 데이터 로드 |
| 크롤링 | GitHub Actions 스케줄/수동 실행 |

### Supabase Redirect URLs

배포 URL이 Redirect URLs에 포함되었는지 재확인하세요.  
비밀번호 재설정 링크 클릭 시 `리다이렉트 URL이 허용되지 않음` 오류가 나면 이 설정을 확인합니다.

### GitHub Actions (크롤링)

Repository → Settings → Secrets and variables → Actions 에서:

- `SUPABASE_URL` (또는 `SUPABASE_SERVICE_ROLE_KEY` 사용하는 URL)
- `SUPABASE_SERVICE_ROLE_KEY`

이 두 Secret이 설정되어 있어야 크롤링 워크플로우가 동작합니다.
